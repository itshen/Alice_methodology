# 第十二章：安全体系

> API Key 是用户最敏感的资产。一旦泄露，损失可能是真实的。

---

## 本地 Agent 的安全挑战

Alice 是本地桌面应用，需要持久化存储：

- 多个 LLM Provider 的 API Keys
- 聊天机器人的 Bot Token（微信、Telegram）
- 对话历史（可能包含隐私信息）
- 用户画像和长期记忆

如果这些数据以明文存在 SQLite 里，任何能访问本地文件系统的程序都能读取。

---

## 两级加密架构

```
用户设置的密码（PIN / 密码短语）
    ↓ PBKDF2（密钥派生函数，10 万次迭代）
主密钥（Master Key，32 字节）
    ↓ 用于初始化加密模块
字段级 AES-256-GCM 加密
    ↓ 用于加密敏感字段
```

**关键点**：用户密码本身**从不存储**。存储的是：
- 随机盐值（用于 PBKDF2）
- 验证哈希（用于校验密码是否正确）

即使数据库文件被窃取，没有用户密码，无法得到主密钥，无法解密任何数据。

---

## 字段级加密，而非全库加密

加密的粒度是字段，不是整个数据库。

**加密的字段**：
- `settings.apiKeys`（LLM Provider 的 Key）
- `settings.botTokens`（微信/Telegram Bot Token）
- 向量数据库里的记忆文本（可选）

**不加密的字段**：
- 对话消息内容（性能考虑，加密/解密大量消息有显著开销）
- 会话元数据（标题、时间、模型 ID）

这是安全与性能的权衡：API Key 泄露会导致直接经济损失，优先保护；对话内容泄露风险相对可控，暂不加密。

---

## 加密标记格式

加密后的值有明确的前缀，方便系统识别：

```
明文：sk-1234567890abcdef
加密后：ENC:v1:base64_iv:base64_ciphertext
```

读取时检查前缀：
- 有 `ENC:v1:` → 需要解密
- 没有 → 明文，直接使用

版本号（`v1`）预留升级空间：未来如果要换加密算法，可以引入 `v2`，同时兼容旧数据。

---

## 解锁流程

```
Alice 启动
    ↓ 检查是否已设置加密（key_meta.json 是否存在）
    ↓ 已设置
显示解锁屏幕（要求输入密码）
    ↓ 用户输入密码
PBKDF2 派生密钥 → 验证是否与存储的哈希匹配
    ↓ 匹配成功
初始化加密模块（cipher.init(masterKey)）
    ↓
重新加载设置（此时能解密 API Keys）
    ↓
初始化 LLM Client（注入解密后的 API Keys）
    ↓
正常进入 Alice
```

**解锁前的 Alice 是"残缺的"**：没有 API Key，无法调用任何 LLM，相当于一个空壳。解锁后 Alice 才"活"过来。

---

## 防止明文覆盖加密数据

一个容易忽视的安全漏洞：

```
情况：
  1. 用户设置了密码，API Keys 已加密
  2. Alice 启动，用户还没解锁
  3. 某个初始化代码读取设置，发现 apiKeys 是加密字符串
  4. 代码 "保存" 了一份设置（可能是某个默认化逻辑）
  5. apiKeys 被空对象覆盖！加密数据丢失！
```

防御措施：当加密模块未初始化时，`setSettings` 检测到设置里有加密字段，直接拒绝写入，抛出"需要先解锁"的错误。

宁可报错，不可静默覆盖。

---

## 记忆的加密

向量数据库里存储的是对话历史的语义化摘要，可能包含敏感信息。

如果加密模块已就绪，写入向量库前加密文本：

```
写入：encrypt(text) → 存入向量库
检索：从向量库取出 → decrypt(text) → 返回明文
```

向量本身（embedding 数值）不加密，因为加密向量会改变其数学特性，导致相似度搜索失效。

---

## 密码安全设计要点

| 要点 | 实现 |
|------|------|
| 密码不存储 | 只存 PBKDF2 盐值 + 验证哈希 |
| 防暴力破解 | PBKDF2 10 万次迭代，每次尝试需要约 100ms |
| 随机盐值 | 每个实例唯一的随机盐，防彩虹表攻击 |
| 随机 IV | AES-GCM 每次加密使用随机 IV，相同明文产生不同密文 |
| 版本前缀 | `ENC:v1:` 支持未来算法升级 |
| 失败安全 | 加密未就绪时拒绝写入，而非静默覆盖 |

---

## 未解决的问题

安全是一个需要持续维护的领域，目前还有一些已知的局限：

1. **对话内容未加密**：内存转储或 SQLite 直接读取可以看到明文对话历史
2. **主密钥在内存里**：进程运行期间主密钥在内存里，内存转储可以提取
3. **备份文件**：如果用户备份了 `~/.alice/` 目录，备份里的数据也需要密码才能解密（这是对的），但用户需要知道这一点

这些是已知的、有意识的局限，在下一个安全迭代版本里逐步解决。
